## Skill Usage Tips

### When to Use Skills
- **Before starting any new feature** → `brainstorming`
- **When implementing complex features** → `writing-plans`
- **When writing code** → `test-driven-development`
- **When debugging** → `systematic-debugging`
- **Before committing** → `verification-before-completion`
- **Before merging** → `requesting-code-review`
- **When receiving feedback** → `receiving-code-review`

### Skill Invocation
In Claude Code, invoke skills using:
```
/skill-name [description of task]
```

Example:
```
/brainstorming Design the combat action system for Gladiator Coliseum
/game-development Implement tick-based combat resolution
/frontend-design Create the arena combat UI with pixel art
```

## Excluded from Demo (Don't Use These Yet)

Per concept.md, these features are explicitly excluded from the demo:
- ❌ Marketplace UI
- ❌ Loot boxes
- ❌ Breeding/forging
- ❌ Token economics
- ❌ Real-money guarantees
- ❌ Advanced ranking ladders

Skills to **avoid until post-demo**:
- micro-saas-launcher (too early for SaaS features)
- ai-wrapper-product (not an AI wrapper)
- shopify-apps (no marketplace yet)
- blockchain-specific marketplace skills
